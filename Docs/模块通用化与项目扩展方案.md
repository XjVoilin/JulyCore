# 活动 / 红点 / 战斗 / 技能 / 引导 — 通用化与项目扩展方案

## 一、设计原则

框架层代码中不应出现：
1. **具体数值语义**：攻击力、防御力、暴击率等
2. **具体玩法机制**：技能冷却、Buff 叠层、伤害公式、奖励槽位等
3. **具体类型枚举**：`AttributeType.Attack`、`SkillType.Ultimate`、`DamageType.Physical` 等

框架只保留：**Id、时间、状态、顺序、生命周期回调、扩展数据容器**。

---

## 二、各模块现状

### 1. 红点（RedDot）— 已通用 ✅

框架只做树结构与数值汇总，业务通过 `SetCalculator`、`BindToSystem`、事件订阅扩展。无需改动。

### 2. 引导（Guide）— 已通用 ✅

数据与表现均由项目通过 `IGuideDataProvider`、`IGuidePresenter`、`IGuideFlowHandler` 注入。无需改动。

### 3. 活动（Activity）— 已重构 ✅

**改造内容**：移除了框架层的「奖励槽位」概念。

框架层只负责：
- 基于时间的状态机（NotStarted / PreAnnounce / InProgress / Ended）
- 活动注册 / 注销 / 查询
- 通用进度读写（`DataPayload` 字符串，项目自行序列化）
- 状态变更事件

项目扩展示例：
```csharp
// 项目自定义进度结构
public class MyActivityProgress
{
    public HashSet<string> ClaimedSlots { get; set; } = new();
    public int Score { get; set; }
}

// 读写
var record = GF.Activity.GetActivityRecord("event_001");
var progress = JsonUtility.FromJson<MyActivityProgress>(record.DataPayload);
progress.ClaimedSlots.Add("slot_1");
GF.Activity.SaveProgressData("event_001", JsonUtility.ToJson(progress));
```

### 4. 战斗 + 技能（Combat + Skill）— 已重构 ✅

**改造内容**：将 RPG 强相关类型全部移除，引入策略模式。

#### 框架层保留

| 类型 | 职责 |
|------|------|
| `ICombatParticipant` | 极简参与者：Id、Team、IsAlive、CanAct、IsAI、ActionPriority |
| `ICombatActionExecutor` | 策略：执行行动（项目实现，含 AI 决策） |
| `ICombatRule` | 策略：胜负判定（项目实现） |
| `ICombatScheduler` | 策略：行动顺序（可选覆盖，默认按 ActionPriority 排序） |
| `ICombatSessionContext` | 只读查询：参与者状态、回合信息 |
| `CombatModule` | 流程骨架：会话生命周期、回合调度、事件发布 |
| `CombatActionPayload` | 通用行动载荷：ActorId + TargetIds + ActionTypeKey + Data |
| `CombatActionResult` | 通用结果：Success + Message + ResultData |

#### 框架层移除

| 已移除类型 | 原因 |
|-----------|------|
| `ICombatEntity` | RPG 属性/技能/Buff/伤害/治疗全耦合 |
| `ISkill` | SkillType/TargetType/DamageType/Cooldown 全是 RPG 概念 |
| `IBuff` | BuffType/BuffCategory/AttributeModifier 全是 RPG 概念 |
| `AttributeType` 枚举 | 具体属性类型 |
| `DamageType` 枚举 | 具体伤害类型 |
| `DamageInfo/DamageResult/HealResult/BuffResult` | RPG 结算数据 |
| `ActionType` 枚举 | 具体行动类型 |
| 业务级事件 | EntityDamaged/Healed/Died、BuffAdded/Removed、SkillUsed、AttributeChanged |

#### 框架层事件（仅流程级）

- `CombatPreparingEvent` / `CombatStartedEvent` / `CombatEndedEvent`
- `CombatPausedEvent` / `CombatResumedEvent`
- `RoundStartedEvent` / `RoundEndedEvent`
- `ActionStartedEvent` / `ActionCompletedEvent`
- `ParticipantJoinedEvent` / `ParticipantLeftEvent`

public class RpgCombatRule : ICombatRule
